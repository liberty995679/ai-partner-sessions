# AI 智能伴侣 — API 接口文档

## 基础信息

- **Base URL**: `http://localhost:8000`
- **认证方式**: JWT Token（通过 HTTP Header `x-token` 传递）
- **Content-Type**: `application/json`

---

## 通用约定

### 通用响应格式

```json
{
  "code": 200,
  "msg": "操作成功"
}
```

### 错误响应

HTTP 状态码 `4xx` / `5xx`，响应体：

```json
{
  "detail": "错误描述信息"
}
```

| 方法 | 路径     | 认证     | 说明               |
|------|----------|----------|--------------------|
| POST | `/login` | 无需认证  | 验证用户名密码，返回 JWT Token |

```json
{
  "username": "string",
  "password": "string"

```

```json

  "code": 200,
  "msg": "登录成功",

}
```

> Token 有效期为 1 小时，之后需在请求头 `x-token` 中携带。

**失败响应** (400):

```json
{
  "detail": "用户名或密码错误"
}
```

### 7. 发送聊天消息（获取 AI 回复）

核心接口：向 AI 伴侣发送消息并获取回复，同时自动保存对话记录。

| 方法 | 路径       | 认证              | 说明                      |
|------|------------|-------------------|---------------------------|
| POST | `/chat_api`| `x-token`（可选）  | 发送消息，调用 DeepSeek 获取 AI 回复 |

**请求体**:

```json
{
  "user_id": "string",
  "message": "string",
  "name": "小薇",
  "prompt": "你叫小薇，是一个善解人意的AI伴侣。",
  "conv_id": 1
}
```

| 字段     | 类型   | 必填 | 说明                                        |
|----------|--------|------|---------------------------------------------|
| user_id  | string | 是   | 用户标识                                     |
| message  | string | 是   | 用户发送的消息内容                            |
| name     | string | 否   | AI 伴侣名字，默认 `"小薇"`                    |
| prompt   | string | 否   | AI 性格设定提示词                             |
| conv_id  | int    | 否   | 对话 ID，传 `0` 或不传时自动使用默认对话       |

**成功响应** (200):

```json
{
  "status": "ok",
  "name": "小薇",
  "reply": "今天心情不错呀！想跟我聊点什么呢？😊"
}
```

| 字段   | 类型   | 说明          |
|--------|--------|---------------|
| status | string | 固定为 `"ok"` |
| name   | string | AI 伴侣名字   |
| reply  | string | AI 回复内容   |

### 11. 获取对话消息

| 方法 | 路径                                  | 认证   | 说明               |
|------|---------------------------------------|--------|--------------------|
| GET  | `/api/conversations/{conv_id}/messages`| 必须   | 获取指定对话的所有消息 |

**路径参数**:

| 参数    | 类型 | 说明    |
|---------|------|---------|
| conv_id | int  | 对话 ID |

**请求头**:

```
x-token: <JWT Token>
```

**成功响应** (200):

```json
{
  "messages": [
    {
      "id": 1,
      "conv_id": 1,
      "role": "user",
      "content": "你好呀",
      "created_at": "2026-07-23 10:01:00"
    },
    {
      "id": 2,
      "conv_id": 1,
      "role": "assistant",
      "content": "你好！今天有什么想聊的吗？😊",
      "created_at": "2026-07-23 10:01:02"
    }
  ]
}
```

| 字段       | 类型   | 说明                                |
|------------|--------|-------------------------------------|
| id         | int    | 消息 ID                             |
| conv_id    | int    | 所属对话 ID                          |
| role       | string | 角色：`"user"`（用户）或 `"assistant"`（AI）|
| content    | string | 消息内容                             |
| created_at | string | 发送时间                             |

---

## 认证机制说明

1. 用户通过 `POST /login` 获取 JWT Token
2. 在需要认证的接口中，将 Token 放入请求头 `x-token`
3. Token 有效期 1 小时，过期后需重新登录
4. JWT 签名算法为 `HS256`，密钥为服务端硬编码的 `SECRET`

### 需要认证的接口

| 接口                                         | 认证方式     |
|----------------------------------------------|--------------|
| `GET/POST /api/conversations`                | `x-token`   |
| `GET /api/conversations/{id}/messages`       | `x-token`   |
| `POST /api/conversations/{id}/messages`      | `x-token`   |
| `DELETE /api/conversations/{id}`             | `x-token`   |
| `POST /chat_api`                             | `x-token`（可选） |
